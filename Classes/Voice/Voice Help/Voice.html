Voice explanation of what Voice is and/or does



Dependencies: VoicePartial


Voice takes an AtsSnapShot and maps the data onto a Resonz filter bank.  Voice can transform from one snapshot to another with additional controls such as a dynamics curve and duration for the transformation.


See also: VoicePartial, AtsSnapShot, AtsFile, Env


Some Important Issues Regarding Voice


This class is under constant development, adding new functionality and undergoing revisions for efficiency.  The Resonz filter bank will be tuned with vocal qualities of a singing voice.


Creation / Class Methods


*new (server, inBus, outBus, snapShot, initamp, bw, onset, curve)

server - The server to play this Voice on. Set to the default Server if none is provided

inBus - The single-channel bus to be read in and sent to the filter-bank

outBus - The output bus of the summed filter bank.

snapShot - An AtsSnapShot from which frequency and amplitude data will be mapped.

initamp - An initial amplitude to start the voice at, to be tuned with bandwidth.

bw - An initial bandwidth of the Resonz filters.

onset - Onset time of the Voice.

curve - A curve to be passed to the Envelopes for bandwidth transformations.


Accessing Instance and Class Variables

group - The group in which the Voice is playing. Each Voice instance creates its own group to store its VoicePartials

inBus - The bus being read in.

currSS - The current-state snapshot.

outBus - The bus to which this Voice is being sent.

transforming - Returns true or false. Another transformation cannot be made until false.

vbw - The Voice's bandwidth

vamp - The overall amp scaler of the Voice.

Instance Methods



transform (targetSnapShot, partialThresh, targetAmp, targetBw, transformTime, curve)

Short prose description of method.

targetSnapShot - A new snapshot to be transformed to.

partialThresh - A threshold in Hz to determine valid candidates for partials to transform to in the targetSnapShot.

targetAmp - The overall amp of the new state of the Voice.

targetBw - The bandwidth of the new state of the Voice.

transformTime - How long the transformation will take. As of now, the transforming flag will take about 5 additional seconds to be set to false for bookeeping.

curve - The envelope curve to be used to transform the bandwith and frequency transformation.


free (releaseTime)

Fade out and free the Voice, including it's Group, voicepartials, control busses and synths.


amp_ (argamp, lag)

Change the Voice's amp over lag time.

bw_ (argbw, argamp, lag, curve)

Change the Voice's bandwidth over lag time, including optional new amplitude and transformation curve.


Examples


// load some snapshots to transform between

~shot = AtsSnapShot.new("My/Ats/File.ats", 0.5, 5, -25.dbamp)

~shot.postData // don't get too crazy with the number of partials

~shot2 = AtsSnapShot.new("Another/Ats/File.ats",0.6, 6, -20.dbamp)

~shot2.postData;

~shot3 = AtsSnapShot.new("YetAnother/Ats/File.ats", 0.6, 13, -15.dbamp)

~shot3.postData

~shot4 = AtsSnapShot.new("StillAnother/Ats/File.ats", 0.6, 6, -20.dbamp)

~shot4.postData


( // start a dummy noise source

Server.default = s = Server.internal.waitForBoot({ s.scope(2,0);

~srcBus = CtkAudio.new(1, 7, s);

{ InGrain.ar(Impulse.kr(30), 

30.reciprocal,

WhiteNoise.ar(0.5))

}.play(s, ~srcBus.bus, fadeTime: 0.1, addAction: \addToHead );

})

)

// create the Voice with it's initial server, inbus, outbus, snapshot, bandwidth, and onset time

v = Voice.new(s, ~srcBus.bus, 0, ~shot,  6.dbamp, 0.005, 2.5, 3);


v.transforming // can only make a transformation if isn't not already transforming

v.vamp // the Voice amp


// change the Voice amp over a lag time

v.amp_(0.5, 4) // v.amp_(newAmp, lagTime)

v.amp_(2, 8)

v.vamp


// change the Voice bandwidth over a lag time

v.bw_(0.0005, argamp: 16.dbamp, lag: 7, curve: 0) // v.bw_(argbw, argamp, lag=5, curve=0)

v.bw_(0.01, argamp: 0.dbamp, lag: 2, curve: 1)

v.bw_(0.1, argamp: -12.dbamp, lag: 8, curve: 2)

// if no amp is provided, it defaults to the present amplitude of the voice

// default lag = 5, default curve is linear

v.bw_(0.05)

// be sure to wait the full transition time or you get jumps in value

v.bw_(0.004, 6.dbamp, 12, 3.5) 

// getter for the voice bandwidth

v.vbw


//transform to another

// v.transform(targetSnapShot, partialThresh=200, targetAmp, targetBw, transformTime=20, curve=0)

v.transform(~shot3, 200, 6.dbamp, 0.005, 6, 2); 

v.transforming // still transforming?


v.amp_(3, 6)

v.bw_(0.01, 0.dbamp, 2, curve: 1)

v.bw_(0.05, -9.dbamp, 6, curve: 3)


// transform to one

v.transform(~shot2, 200, 0.dbamp, 0.005, 8);

v.currSS.path // which snapshot are we on?


// another...

v.transform(~shot4, 200, 9.dbamp, 0.001, 7);


//back to original

v.transform(~shot, 200, -6.dbamp, 0.01, 15);


// use AtsSnapShot functionality to shift pitch or frequency

// transform to ~shot, but transposed up a 3rd, set the threshold accordingly

v.transform(~shot.transposeAndCopy(3), 400, 6.dbamp, 0.005, 10);


v.free(6) // exponential decay by default

~srcBus.free; // free the source bus from the dummy noise
