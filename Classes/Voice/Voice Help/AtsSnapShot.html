AtsSnapShot A snapshot of an AtsFile averaged over specified frames.



Dependencies: AtsFile


AtsSnapShot extracts frequency and amplitude information from a specified point in an AtsFile.  The data will be averaged over a specified number of consecutive points centered at the pointer effectively giving you steady-state partial information from the soundfile analyzed by ATS.  Partials under ampThresh will be excluded, and the surviving partial amplitudes are normalized. Additional manipulation of the data is possible, such as pitch or frequency shifting.


See also: AtsFile


Creation / Class Methods


*new (path, pointer, framesToAverage, ampThresh, server)

path - The path to your AtsFile to be parsed.

pointer - Where in the AtsFile to take the snapshot, from 0 to 1. Default value is 0.5.

framesToAverage - The number of frames to average frequency and amplitude data. Default value is 3.

ampThresh - An amplitude threshold below which partials are rejected. Surviving partials are then normalized to 1. Default value is nil. Other information.

server - Default value is nil, in which case server will be set to the default.

// inline example

b = AtsSnapShot.new("My/Ats/File.ats", 0.75, 5, -30.dbamp)

b.postData; // see the results of your snapshot



Accessing Instance and Class Variables

freqs - Returns the frequencies of the snapshot in ascending partial order.

amps - Returns the amplitudes of the snapshot in ascending partial order.


origFreqs - Returns the original frequency information before any transposing, etc. was done to the data.

origAmps - Returns pre-normalized amplitude data of the snapshot

Instance Methods



farm (pointer, framesToAverage, ampThresh)

AtsFile is farmed for data upon creation of the snapshot, but may be re-farmed with new arguments if the initial results aren't desireable. The newly-farmed data persists.

pointer - Explanation of pointer. Default value is nil. Other information.

framesToAverage - Explanation of framesToAverage. Default value is nil. Other information.

ampThresh - Explanation of ampThresh. Default value is nil. Other information.

b = AtsSnapShot.new("My/Ats/File.ats", 0.75, 5, -30.dbamp)

b.postData; // didn't like the results?

b.farm( 0.7, 3, -45.dbamp );

b.postData; // better?


postData

Post the basic data of the snapshot for quick reference.



test (bw, mul, dur, server)

Test the snapshot with a granulated noise source being filtered by Resonz filters that take the snapshot results as parameters.

bw - Bandwidth of the Resonz filters. Default value is 0.005. Other information.

mul - Amplitude of the Resonz test. Default value is 1.5. Other information.

dur - Length of the test. Default value is 5. Other information.

server - Optional server argument to test the snapshot on.


b = AtsSnapShot.new("My/Ats/File.ats", 0.75, 5, -30.dbamp);

Server.default = s = Server.internal.waitForBoot({ s.scope; b.test(dur: 5, server: s) });

b.test;


transpose (transposeVal, semitone)

Transpose the frequencies of the snapshot by a scalar or by semitone. This is a desctructive change to the data, changing the state of the instance. You may reset to original frequencies by calling .reset on the snapshot.

transposeVal - A scaler or semitone value to transpose the data by.

semitone - A boolean. True for semitone transposing, false for scalar values. Default is true.

// inline example

g = myAtsSnapShot.transpose(5, true); // transpose the partials up a fifth

g.freqs; // freqs are permanently transposed in this snapshot now


transposeAndCopy (transposeVal, semitone)

Like .transponse but returns a new snapshot, leaving the original as it was.

transposeVal - A scaler or semitone value to transpose the data by.

semitone - A boolean. True for semitone transposing, false for scalar values. Default is true.

// inline example

newShiftedSShot = myAtsSnapShot.transposeAndCopy; // returns a new snapshot

newShiftedSShot.freqs; // a new snapshot with transposed freqs

myAtsSnapShot.freqs; // the original snapshot is unaffected


shift (shiftHz)

Like transpose, but frequency shifting each partial by shiftHz.



shiftAndCopy (shiftHz)

Returning a new snapshot whose frequencies are shifted equally by shiftHz.



reset

Restore the snapshot to its orginal, or most recently farmed state (undoing any frequency shifting or transposing).



snapFreqs

Returns the partial frequencies of the snapshot. Same as mySnapShot.freqs.



snapAmps

Returns the partial amplitudes of the snapshot. Same as mySnapShot.amps.



numSnapPartials

Returns the number of partials in the snapshot.

isolatePartials (first, last)

Returns a array of the partial freqs and amps from the snapshot from first desired partial to last desired partial. The returned data is formatted [ [ freqs ], [ amps ] ].



isolatePartialFreqs (first, last)

Same as isolatePartials, but only returns the freqs within the range of first desired partial to last desired partial.



isolatePartialAmps (first, last)

Same as isolatePartials, but only returns the amps within the range of first desired partial to last desired partial.



normalizeAmps (normValue)

Normalize the amplitudes to a an amplitude of normValue. Recall the snapshot results are already normalized to 1.



freqAmpPairs

Returns an array of frequency/amp pairs of the snapshot.



Examples


// Basic functionality

// Sending AtsSnapShot only a file name gives you a default snapshot at time 0.5, averaging 3 frames, with an amp threshold of -60.dbamp

b = AtsSnapShot.new("Your/Ats/File.ats")

b.postData; // more info

b.numSnapPartials; // how many partials did we get?


// alternatively, create a new AtsSnapShot with your own parameters:

b = AtsSnapShot.new("Your/Ats/File.ats", 0.75, 5, -30.dbamp)

b.postData;

// re-farm it for new values - this permanently updates the instance's variables

b.farm(0.25, 7, -60.dbamp); //pointer, framesToAverage, ampThreshold

b.postData

//same

b.snapFreqs

b.freqs 

//same

b.snapAmps

b.amps

// what was the amp threshold for this snapshot?

b.thresh


// transpose frequencies

b.freqs // original

b.transpose(3, false).freqs; // transpose x 3

b.reset.freqs // original

b.transpose(3, true).freqs; // by semitone

b.reset.freqs // original


// frequency shift values

b.freqs // original

b.shift(500).freqs; // shifted

b.reset.freqs // back to original


////////////

// Transposing, returning a new AtsSnapShot instance

////////////


c = b.transposeAndCopy(5)

b.postData

c.postData // amp data intact

b.freqs

c.freqs

b.numSnapPartials.do({ | i |

b.freqs.floor.includes( (c.freqs[i] * -5.midiratio).floor ).postln

}) 

)


c = b.shiftAndCopy(25);

b.freqs

c.freqs


( //test

b.numSnapPartials.do( { | i |

b.freqs.floor.includes( (c.freqs[i] - 25).floor ).postln

})

)
